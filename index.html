    Trip Optimizer

      Developed an Android-based Application that generates optimized travel routes based on user's selection of places and restaurants.

      The application is connected to the Relational Database System (MySQL) on AWS server and uses API calls on backend through AWS Lambda to perform HTTP requests and algorithms. Users select places on the database that contains name, address, business hours, and ratings from Yelp and TripAdvisor. All information is scraped from both TripAdvisor and Yelp and merged together to provide user accurate and less-missing information.

      On AWS server, it runs sub-optimal traveling salesman algorithm to generate efficient itinerary in terms of time that connects all selected places. It also includes nearest restaurants among user's selection during lunch/dinner time on the itinerary.

      Team project for CS411: Database System course at the University of Illinois.
